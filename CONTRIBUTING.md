# Contributing

Thanks for your interest in contributing to Noolava!

How to contribute
1. Fork the repository and create a feature branch:
   git checkout -b feat/your-short-description
2. Make changes, keep commits focused and documented.
3. Push to your fork and open a Pull Request to the upstream repository.

Code style
- Keep markup semantic and accessible.
- Prefer vanilla JS utilities, avoid adding heavy dependencies.
- Document new UI behavior in docs/USAGE.md.

Reporting issues
- Use GitHub Issues — include steps to reproduce, browser/OS, and screenshots when relevant.
